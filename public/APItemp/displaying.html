Displaying Memos

When memos have been pushed to a users account on memolane they will be rendered based on two templates defined when registering the app on the my apps page

For each registered application, 2 separate templates are needed, one for a small version of a memo and one for the large version (although in some cases having the identical templates for both makes sense)

The template language used is based on mustache which is in essence just plain text with the addition of placeholder tokens for various pieces of data and the option to make these placeholders optional depending on whether a particular element is present in the data that makes up the memo being displayed.

For describing a memo, Memolane uses a small block of html with one or more of these "placeholders" added.

Like the data format, the templates can utilize a number of building blocks that can be be added to a template as needed. The building blocks used should match the data that is pushed as part of the memo data or should make blocks optional depending on whether that piece of data is available in the data or not.

An example of a very simple template for a memo that just displays a bit of short text (such as a Twitter memo) could be:

<div>{{{title}}}</div>

A slightly more involved example where there is always text to show but where there might or might not be image and author elements included in the data for a particular memo would be:

						{{#author}}
						  <div>
						    <b> From {{{author}}}: </b>  
						  </div>
						{{/author}}
						{{{title}}}
						{{#image}}
						  <div>{{{image}}}</div>
						{{/image}}

In these examples {{{_element_name_}}} means that this element should always be shown (this will cause errors if that element is not present in particular memo) and a construct like {{#_element_name_}} … {{/_element_name_}} means that everything between these two blocks should only be shown if the memo data contains and that particular element. So for instance...

							{{#author}}
							  <div>
							    <b> From {{{author}}}: </b>   
							  </div>
							{{/author}}

…will only add anything to the memo if the memo data actually includes an "author" element.

The types available for use in templates matches the fields available for memo data and are as follows:

{{{title}}}
{{{description}}}
{{{text}}}
{{{geo}}}
{{{image}}}
{{{video}}}
{{{author}}}
{{{event}}}

Each of these might be shown slightly differently on small and expanded memos respectively. Check out the requirements for each of these data formats.
