Data format of a Memo

Introduction

When pushing a memo to Memolane, it has to conform to a set of rules in order for it to show up correctly. The data making up a memo consists of a few mandatory fields and a number of "building blocks" which can be included as needed.

Required fields

The required fields are "id", "created_at", and "privacy".

"id" is a string generated by the posting entity. The constraint on this is that it has to be unique for the item represented by the memo for the user on whose behalf it is being posted to Memolane. This id can then later be used to retrieve the posted memo and to update it as needed (pushing privacy to text changes or any other updates made to the original content)

"created_at" is the unix timestamp representing the time the memo will be shown on the timeline.

"privacy" is the privacy setting for which this memo will initially be shown on Memolane. This needs to conform to the Memolane privacy levels of either "private", "public", or "friends". It is up to the developer to determine how the source privacy levels map tot he Memolane values. If in doubt, post all content as private. The user is able to change this on a per memo level later if he wishes to share some of this content with others in the context of Memolane.

Building blocks

The following is a description of the optional building blocks available for building a memo. Each of these blocks represents a certain visual element and has a number of required parts and can have one or more optional parts.

For a visual reference as to how this information is represented on Memolane, have a look at our template types documentation.

title

The title element represents the main title or name of the Memo. The "text" string will always be shown while the optional "thumbnail" is not currently shown (if present in the title element) but this is likely to change in the future.

required:
text String
optional:
thumbnail String

Example JSON

							"title":{
							"text" : "A memo with an image"
							}

description

The description element represents a textual description of the memo, such as the description of a photo or other media element.

required:
text String

Example JSON

							"description":{
							"text" : "A cool day at the warm beach"
						}

text

The text element is used for larger chunks of textual content and html. This is useful for blogs and the like, but it is also the only element allowing html to be rendered, so it can be used to embed custom flash players and the like.

required:
text String

Example JSON

						"text":{
						"text" : "WOW I just found out that...."
						}

geo

A geo elements represents points on a map, a section of lines on a map or a combination of the two. It can be used for such things as displaying the location of geo-tagged info, "chekins" on location based services or "tracks" such as gps info about an bike ride or the route of cross country drive.

optional:
polyline Array of polyline point elements(min 2 points required if present)
poi Array of poi elements

A polyline point element has the following structure

required:
lat Float representation of latitude
poi float representation of latitude
optional:
ele Elevation of the point
timestamp Unix timestamp when this point was recorded
A poi (point of interest) element has the following structure
required:
name String the name of the point of interest
lat Float representation of latitude
lon Float representation of longitude
optional:
description String description of the POI
time Unix timestamp when this point was recorded
address String describing the address of the POI

Example JSON

						"geo":{
						"polyline":[ 
						{ 
						"lat":"50.031944", 
						"lon":"8.527778" 
						}, {
						"lat":"55.608056",
						"lon":"12.633333"
						}
						],
						"poi":[
						{ 
						"name":"Frankfurt am Main FRA",
						"description":"Departure (United Airlines UA 9074)",
						"time":1298293500,
						"lat":"50.031944",
						"lon":"8.527778"
						}, {
						"name":" Copenhagen CPH",
						"description":"Arrival (United Airlines UA 9074)",
						"time":1298298300,
						"lat":"55.608056",
						"lon":"12.633333"
						}
						]
						}

image

An image element represents an image to be shown as part of a memo. The image comes with 3 different urls for various purposes, but if these are not available from the source, they can all be set to the same value.

required:
url String
thumbnail String
original String

Example JSON

						"image":{
						"url":"http://media.tumblr.com/tumblr_lhr1bm7DG31qbwaa2.jpg",
						"thumbnail":"http://media.tumblr.com/tumblr_lhr1bm7DG31qbwaa2.jpg",
						"original":"http://media.tumblr.com/tumblr_lhr1bm7DG31qbwaa2.jpg"
						}

video

required:
player String
thumbnail String

Example JSON

						"video":{
						"video_player":"  \n",
						"thumbnail":"http://i.ytimg.com/vi/ImFHrS8YCaM/hqdefault.jpg"
						}

Notes

In embedded video players, if possible, replace height and width values with __WIDTH__ and __HEIGHT__ respectively as this will ensure that the video gets scaled to the correct size of the memo when it is displayed.

author

An author element can be used to show that a particular piece of content is not by the Memolane user himself but rather created by someone else and to give credit to the actual creator. This is useful if content has been added as a "favorited", "liked", "recomended" or similar.

required:
name String
optional:
id String
year Fixnum
url String

Example JSON


						"author":{
						"name":"ptreeofficial",
						"url":"http://www.youtube.com/user/ptreeofficial"
						}

event

An event element represents a calendar event.

required:
location String
ends_at Fixnum
optional:
address Hash
attending Fixnum (1 or 0)

Example JSON

						"event":{
						"location":"Humboldt-Universität, Berlin, Germany",
						"ends_at":1313204400,
						"attending":"attending"
						}

Notes

An event object uses the "created_at" timestamp of the memo as it's start time, which is why the start time is not specified in the "event" element itself.

Example Memos

Here are two examples of the complete JSON date of two valid example memos. These are both memos from the MemoNotes demo application:

A simple memo with title and text

						{
						"id":1307728133,
						"created_at":1307728133,
						"privacy":"friends",
						"title":{
						"text":"A simple memo"
						},
						"text":{
						"text":"The bread text of this very basic example of valid memo json"
						}
						}

A memo with title, text and image

						{
						"id":1307728317,
						"created_at":1307728317,
						"privacy":"public",
						"title":{
						"text":"A memo with an image"
						},
						"text":{
						"text":"This memo has both a text element and an image."
						},
						"image":{
						"url":"http://media.tumblr.com/tumblr_lhr1bm7DG31qbwaa2.jpg",
						"thumbnail":"http://media.tumblr.com/tumblr_lhr1bm7DG31qbwaa2.jpg",
						"original":"http://media.tumblr.com/tumblr_lhr1bm7DG31qbwaa2.jpg"
						}
						}
